網站架設

不用虛擬主機,不用付費主題,免費架設出一個網站！

By 老湯隨談 26 October 2020

在這個網路非常發達的世代,個人網站是一個非常好展現想法的媒介.

       我相信很多人應該都有過想要架設網站的想法吧？但是當真正去做的時候才發現自己不會寫 HTML 也不會寫 CSS ,甚至完全沒有接觸過任何程式語言.不過這也沒有關係,還有很多平台可以選擇,比如說可以使用 world press 或是 Wix ,但是接觸後才發現,也許並不是很適合自己.想要一個好一點的主題,需要花錢;想要讓網站一直在線上,需要花錢購買虛擬主機.

       如果你只是想要一個簡簡單單的網站,可以放一些自己想分享的事情,可以記錄生活周遭的各種奇聞趣事,那我相信你現在所看的這篇文章將會對你有很大的幫助.因為我今天要介紹的就是一種方便的架網站方法,他不需要學會去寫很複雜的程式,也不需要花錢去購買付費主題或是虛擬主機.

如何免費架設網站？

       在架設網站之前,你會有一些需要用到的東西,推薦預先全部準備好,這樣會讓你架設網站的流程變的比較順暢,以下為所需要的東西.

  • Github 賬號
  • Visual Studio Code

我們這次將會使用 Github Page + Hugo 的方式來架設網站,順帶一提,本網站（隨緣記）也是使用這個方式架立的.如果準備好了以上的東西,那就可以準備開始咯！

下載Hugo,挑選主題

       打開搜尋引擎,去搜索 Hugo ,理論上來說,第一個就會是我們需要的結果.進入網頁後,將網頁拉到最下方找到: See the Getting Started Guide for other methods 並且點擊進去.點擊進去之後再去尋找 Hugo Releases 並且點擊進去,然後找到 hugo_extended ,選擇自己相對應的系統,並且下載.下載完後解壓縮,可以得到一個Hugo.exe的程式,那個程式要留著,之後會用到.

       下載好了Hugo以後,接下來我們就可以來選擇我們喜歡的主題了,回到Hugo的官方網站並且點擊 Themes ,在這裡你可以看到非常多的主題,而本網站（隨緣記）使用的主題是 Liva Hugo

       我以 Liva Hugo 為例,下載下來以後,解壓縮,把其中的 exampleSite 資料夾拖出來放置桌面（或任何你喜歡的地方）,然後在其資料夾裡建立一個新的資料夾,檔名叫做 themes .創立好後,回到剛剛下載好的資料夾（ liva hugo-master ）將其改名為 liva hugo 且剪下他並且放入 themes 裡面.然後在把最一開始下載的 Hugo.exe 檔案放入 examplesite 資料夾內,到這裡檔案就算是建立完了！

       到了這邊,就已經可以叫出自己的網站來看看了.我們在 exampleSite 的資料夾內,叫出命令提示字元,在其輸入hugo server,然後就可以得到一串網站 (localhost:1313) ,將這串網址複製到瀏覽器上就可以看到我們的網頁啦！不過值得注意的是,在這邊得到的網址只有你自己可以使用,其他人是無法用的,所以我們後面還有許多步驟要做.

如何修改文章

       想要修改文章,就必須使用相對應的工具了,這邊我推薦使用 Visual Studio Code ,簡稱 VS Code. 在剛打開 VS Code 的時候,推薦大家可以先安裝一些插件,讓我們可以更順利的完成任務.

       都弄好了以後,就可以在 VS Code 上面打開我們的 exampleSite 的資料夾, 在裡面找到 Content 資料夾,在Content資料夾內可以看到 blog 的資料夾,我們的文章都會儲存在這裡.

       隨便點開一篇 post ,可以看到裡面有許多的預設內容,這些內容大部分都是以Markdown的語法來呈現.所以在編輯自己想要的文章時,也可以參考一下相關的語法,讓文章變得排版好看一點.

如何讓網站上線？

       都完成以後,就要準備將網頁上傳到 Github 了,這樣才可以讓別人可以透過網址來看你的網頁.所以接下來,我就要來講解如何將自己的網頁上傳上去,

  1. 在Github上面建立一個專案,專案名稱需要為：自己的賬號名稱.github.io
  2. 在 exampleSite 的資料夾內叫出命令提示字元,並且在上面輸入 Hugo .
  3. 在 exampleSite 資料夾內找到 public 資料夾
  4. 將 public 資料夾內所以文件上傳到你所建立的專案那邊

到這邊,網站就已經成功上線了,網站的網址就會是你的 Github賬號名稱.github.io ,可以在瀏覽器上試試看.如果頁面顯示為404,那則需要耐心等待一段時間,網站就會順利上線了,如果一直都會404,那就是有哪個步驟做錯了.

如何修改上線網站的內容

       這邊就以修改文章（blog）為例.首先去到 exampleSite\content\blog 裡面去更改或是新增你的文章.這邊要非常注意的是：不是在 public 資料夾內作業！ 修改完後,再回到exampleSite資料夾,叫出命令提示字元,輸入 hugo .複製 public 資料夾內的檔案,重新上傳 Github 你所建立的專案那邊.這樣就可以了,線上的網站也會進行更改.修改其他的東西也是相同的道理.
